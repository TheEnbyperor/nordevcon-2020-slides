📜 A brief history of JS 📜

  • First appeared in 1995
  • Doesn't actually have that much to do with Java
  • Supported by every major browser
  • A bit on the slow side
  • New features prove problematic
Start of 1995: netscape wanted to add interactivity to their browser Collaborated with Sun to embed Java and Brendan Eich to embed Scheme (a dialect of Lisp) Decided that Eich should devise an entirely new programming language. Released in September as LiveScript, renamed to JavaScript 3 months later. Supported in every major browser due to Microsoft copying the work of Netscape in Internet Explorer Interpreter driven, so could be quicker. As new features are released a need for polyfills arises due to the need to support olde browsers.

🆕 WASM 🆕

The new kid on the block

But why?

  • Native or near native speed
  • Support for any non garbage collected language (gc support incoming)
  • OS level threading
  • Still sandboxed, so can't be too naughty
  • JavaScript is sticking around

Support?

Development started in 2015. MVP in March 2017. Support in the 4 major browsers in November. asm.js, which was a earlier attempt can be comiled from WASM
